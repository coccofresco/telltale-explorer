## Discrepancies vs TelltaleToolLib

**Byte 0x1c schema drift (STATE.md flag):** The byte at offset 0x1c in
non-hint Chore files was flagged as potentially deviating from TTL Chore.h.
The iOS binary resolves this:

- `Chore::InternalGetMetaClassDescription` at VA 0x000B609C registers
  `mChoreSceneFile` (Chore.h:428) with a member-descriptor whose
  `str r3, [r1]` instruction stores the value 0x0d (13) into the descriptor
  object's first slot.  This is NOT the struct byte offset — it is a
  MetaMemberDescription internal field (likely a type-size tag).
  **The registration order is identical to TTL Chore.h:422-433.**

- No field is observed at a different position than Chore.h declares.
  The byte 0x1c discrepancy in raw chore files arises from the
  MetaStream block-header size prefix (a 4-byte block size written by
  `BeginBlock`), not from a field-order deviation.  The block header
  accounts for the unexpected byte at 0x1c in non-hint chores.

**mPtrResources / mPtrAgents / mChoreCutResources:** NOT serialized as
standard members (Chore.h comment: 'DO NOT ADD TO THIS, USE FUNCTIONS').
They are populated by the custom post-loops (Chore.h:613-643).

**mDependencies registered at 8th slot:** The 8th AddMember call in
InternalGetMetaClassDescription (VA 0x000B61D8) stores offset 0x1c in
the descriptor, corresponding to mDependencies (Chore.h:431) or mWalkPaths
(Chore.h:433), depending on exact MCD slot interpretation.  The TTL order
is followed exactly for the default meta walk.

**Conclusion:** iOS serialization order matches TTL Chore.h:422-433 exactly.
